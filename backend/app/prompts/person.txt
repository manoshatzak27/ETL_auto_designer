The person table maps one row per patient. Key transformation rules:

1. person_id: two modes depending on the config:
   a) Normal mode (auto_increment == false or absent): cast the source ID column using the `transform` field in the config:
      - "int_float" (default): int(float(value)) — for IDs stored as floats like "1.0"
      - "int": int(value) — for IDs stored as plain integers like "1"
      - "str": str(value) — keep as string
      IMPORTANT: always use the transform value from the config JSON, not a hardcoded cast.
      person_source_value = raw source ID value (string).
   b) Auto-increment mode (auto_increment == true): assign sequential integers starting from 1 for each output row (do NOT read any source column). person_source_value = string of that sequential integer.
2. person_source_value: see rule 1 above — depends on the chosen mode
3. gender_concept_id: use the value_map from config (e.g. {"1.0": 8507, "2.0": 8532} where 8507=Male, 8532=Female)
4. gender_source_value: raw source gender value
5. year_of_birth, month_of_birth, day_of_birth: parsed from date_of_birth column using the `date_format` value from the config (e.g. year_of_birth.date_format). IMPORTANT: always use the exact format string from the config — do NOT default to "%Y-%m-%d" unless that is what the config actually contains.
6. race_concept_id: 0 unless mapped
7. ethnicity_concept_id: 0 unless mapped
8. All *_source_concept_id fields: 0
9. FILTER: skip rows where required_source_cols are missing or empty

## PROVIDER ID LOOKUP

The PROVIDER CONFIG section (injected below) contains provider_name_col (the source column
that identifies the provider per row).

If provider_name_col is configured (non-empty):
  - Load ETL_OUTPUT_DIR/provider.csv (semicolon-delimited).
  - Build dict: provider_lookup = {str(row["provider_name"]): int(row["provider_id"]) for each row}
  - If provider.csv does not exist, set provider_lookup = {} and continue — do not abort.
  - For each person row, set provider_id = provider_lookup.get(str(row[provider_name_col]), None)

If provider_name_col is empty or absent in the config, set provider_id to NULL.

## LOCATION LOOKUP

The LOCATION CONFIG section (injected below) contains the person address columns:
  address_1_col, address_2_col, city_col, state_col, zip_col, county_col, country_source_value_col

For each source row, compute location_source_value exactly as location.py does:
  - Collect the values of each configured person address column (skip unconfigured ones).
  - Join with " | " separator, omitting null/empty values.
  - This string is the lookup key into location.csv.

Location lookup steps (only when at least one person address column is configured):
  - Load ETL_OUTPUT_DIR/location.csv (semicolon-delimited).
  - Build dict: location_lookup = {row["location_source_value"]: row["location_id"]}.
  - If location.csv does not exist, set location_lookup = {} and continue — do not abort.
  - Set location_id = location_lookup.get(computed_location_source_value, None).

If no person address columns are configured in the LOCATION CONFIG, set location_id to NULL.

Output columns (in this order):
person_id, gender_concept_id, year_of_birth, month_of_birth, day_of_birth, birth_datetime,
race_concept_id, ethnicity_concept_id, location_id, provider_id, care_site_id,
person_source_value, gender_source_value, race_source_value, race_source_concept_id,
ethnicity_source_value, ethnicity_source_concept_id
