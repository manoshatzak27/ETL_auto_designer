|---|---|---|---|

| provider_name | varchar(255) | No | Map from `provider_name_col` if provided. Truncate to 255 chars. |
| npi | varchar(20) | No | National Provider Identifier. Map from `npi_col`. Truncate to 20 chars. |

| year_of_birth | integer | No | Map from `year_of_birth_col`. Parse as integer. Null if not configured or parse fails. |
| gender_concept_id | integer | No | Look up per-row from `gender_concept_value_map` (a dict in the config) using the row's raw value of `gender_source_value_col`. If the map is absent or the value is not a key in the map, use `gender_concept_id_default` (default 0). |
| provider_source_value | varchar(50) | No | Map from `provider_source_value_col`. Truncate to 50 chars. |
| specialty_source_value | varchar(50) | No | Map from `specialty_source_value_col`. Truncate to 50 chars. |
| specialty_source_concept_id | integer | No | Always 0. |
| gender_source_value | varchar(50) | No | Map from `gender_source_value_col`. Truncate to 50 chars. |
| gender_source_concept_id | integer | No | Always 0. |

## specialty_concept_id lookup

If `specialty_concept_value_map` is present in the config, build a Python dict from it at script startup.
For each provider row, read the raw value from `specialty_source_value_col` and look it up in that dict:

```python
specialty_map = config.get("specialty_concept_value_map", {})
# Per row:
raw_specialty = str(row.get(specialty_source_value_col, "")).strip()
specialty_concept_id = specialty_map.get(raw_specialty, 0)
```

If `specialty_concept_value_map` is absent or the value is not found, set `specialty_concept_id` to 0.

## gender_concept_id lookup

If `gender_concept_value_map` is present in the config, build a Python dict from it at script startup.
For each provider row, read the raw value from `gender_source_value_col` and look it up in that dict:

```python
gender_map = config.get("gender_concept_value_map", {})
gender_default = config.get("gender_concept_id_default", 0)

gender_concept_id = gender_map.get(raw_gender, gender_default)
```

If `gender_concept_value_map` is absent or the value is not found, use `gender_concept_id_default` (0 if not set).

## Deduplication

Deduplicate by `provider_source_value` when configured. If multiple source rows have the same `provider_source_value`, emit only one PROVIDER row (first occurrence). If `provider_source_value_col` is not configured, emit one row per source row.

## care_site_id lookup

Load `ETL_OUTPUT_DIR/care_site.csv` if it exists. Build a dict mapping `care_site_source_value → care_site_id`. For each provider row, look up the value of `care_site_source_value_col` in this dict. If no match or file absent, set `care_site_id` to None.
