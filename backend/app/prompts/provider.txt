## specialty_concept_id lookup

If `specialty_concept_value_map` is present in the config, build a Python dict from it at script startup.
For each provider row, read the raw value from `specialty_source_value_col` and look it up in that dict:

```python
specialty_map = config.get("specialty_concept_value_map", {})
# Per row:
raw_specialty = str(row.get(specialty_source_value_col, "")).strip()
specialty_concept_id = specialty_map.get(raw_specialty, 0)
```

If `specialty_concept_value_map` is absent or the value is not found, set `specialty_concept_id` to 0.

## gender_concept_id lookup

If `gender_concept_value_map` is present in the config, build a Python dict from it at script startup.

## care_site_id lookup

The injected CARE SITE CONFIG section contains `care_site_name_col` — the source column that identifies the care site per row.

Load `ETL_OUTPUT_DIR/care_site.csv` (semicolon-delimited) if it exists.
Build a dict mapping care_site_name → care_site_id:

```python
care_site_lookup = {str(r["care_site_name"]): int(r["care_site_id"]) for r in care_site_rows}
# Per row:
care_site_id = care_site_lookup.get(str(row[care_site_name_col]), None)
```

If no match, file absent, or `care_site_name_col` is not configured, set `care_site_id` to None.

## PROVIDER_SOURCE_VALUE — AUTO-COMPUTED

provider_source_value is NOT read from any source column. Build it as:

```python
provider_source_value = (str(care_site_id) + ' | ' + str(row[provider_name_col]))[:50]
```

where care_site_id is the resolved OMOP care_site_id for this row (use the string 'None' if care_site_id is None).
IMPORTANT: cast every value to str() before joining — columns may be integers.
Truncate to 50 chars. Use this composite value as the deduplication key.

## Deduplication

Deduplicate by the auto-computed provider_source_value. If multiple source rows have the same provider_source_value, emit only one PROVIDER row (first occurrence).
