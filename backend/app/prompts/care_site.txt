Generate a Python ETL script that transforms the source data into the OMOP CARE_SITE table.
The script must be entirely driven by the USER CONFIGURATION provided below.

## CONFIG KEYS AND HOW TO USE THEM

Read the USER CONFIGURATION JSON. Each key maps directly to a behaviour:

- care_site_name_col: source column name → care_site_name (varchar 255).
  If empty string or absent, set care_site_name to NULL for all rows.

- care_site_source_value_col: source column name → care_site_source_value (varchar 50).
  Used as an additional deduplication key (see step 4). If empty or absent, omit from dedup.

- place_of_service_concept_id: integer constant (or null) → same field on every output row.
  Copy this value as-is into every CARE_SITE record. If null, write NULL.

- place_of_service_source_value_col: source column name → place_of_service_source_value (varchar 50).
  If empty or absent, set to NULL.

- location_source_value_col: source column name whose values are used to look up location_id
  from the already-generated location.csv.
  If empty or absent, set location_id to NULL for all rows.

A column is "not configured" when its value is an empty string "" or the key is missing entirely.
Never read a source column that is not configured. Never invent default column names.

## IMPLEMENTATION STEPS

1. Read the source file (ETL_SOURCE_PATH) with the correct delimiter and encoding.

2. Location lookup (only if location_source_value_col is configured):
   - Load ETL_OUTPUT_DIR/location.csv (semicolon-delimited).
   - Build dict: location_lookup = {row["location_source_value"]: row["location_id"]}.
   - If location.csv does not exist, set location_lookup = {} and continue — do not abort.

3. For each source row, build a candidate CARE_SITE record:
   - care_site_name              ← row[care_site_name_col]               if configured, else NULL
   - care_site_source_value      ← row[care_site_source_value_col]       if configured, else NULL
   - place_of_service_concept_id ← constant from config                  (integer or NULL)
   - place_of_service_source_value ← row[place_of_service_source_value_col] if configured, else NULL
   - location_source_value_raw   ← row[location_source_value_col]        if configured (for lookup only)
   - location_id                 ← location_lookup.get(location_source_value_raw, None)

4. Deduplication — MANDATORY:
   Build the full list of candidate rows first, then deduplicate BEFORE assigning care_site_id.
   Two records are duplicates if they share the same care_site_name (case-insensitive, after
   stripping whitespace). Apply in this priority order (keep first occurrence):

   a. If care_site_name_col is configured:
        Always deduplicate on care_site_name. Two rows with the same care_site_name
        (case-insensitive strip) are the same Care Site — emit only the first one.
   b. If care_site_source_value_col is also configured:
        Additionally deduplicate on care_site_source_value so that two records with the same
        source value are never emitted even if their names differ.
   c. If care_site_name_col is NOT configured but care_site_source_value_col is:
        Deduplicate on care_site_source_value only.
   d. If neither is configured:
        No deduplication is possible; emit all rows.

   Implementation: normalise care_site_name with .str.strip().str.lower() into a helper column,
   call drop_duplicates on the appropriate subset, then drop the helper column.

5. Assign care_site_id: sequential integers 1, 2, 3, … after deduplication.

6. Write output to ETL_OUTPUT_DIR/care_site.csv (semicolon-delimited, UTF-8).
   Output columns in this exact order:
   care_site_id, care_site_name, place_of_service_concept_id, location_id,
   care_site_source_value, place_of_service_source_value

## ROBUSTNESS RULES

- Wrap per-row logic in try/except; log errors and skip the row.
- Do NOT abort if location.csv is missing — write NULL for location_id.
- Truncate care_site_name to 255 chars, care_site_source_value / place_of_service_source_value
  to 50 chars if the source value is longer.
- Print progress: "Writing care_site.csv ... done (N records)"
- Guard: if __name__ == '__main__': main()
